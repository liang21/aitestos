# 智能测试管理平台 · 产品需求文档（PRD）

**版本**：2.1
**日期**：2026-02-24
**产品经理**：10年产品专家
**状态**：正式发布

------

## 1. 产品概述

### 1.1 背景

在软件研发过程中，测试用例的编写与维护占据大量人力成本，且高度依赖测试工程师的个人经验。产品需求文档（PRD）、Figma设计稿等源头文档与最终测试用例之间存在断层，导致用例覆盖不全、需求变更后用例更新滞后、需求与测试之间难以追溯。
随着AI技术的成熟，利用大语言模型自动将需求文档转化为测试用例已成为可能。本产品旨在打造一个 **AI 驱动的智能测试管理平台**，通过 RAG（检索增强生成）技术将非结构化产品文档转化为结构化测试用例，并与完整的测试生命周期管理（计划、执行、结果分析）无缝集成，实现“需求即测试、测试即代码”。

### 1.2 目标

- **提升用例编写效率**：减少 80% 的手动用例编写工作量，测试设计周期缩短 50%。
- **保证用例质量**：通过 AI 生成 + 人工审核机制，确保用例覆盖率 ≥90%，异常场景丰富度超越纯人工。
- **实现需求可追溯**：每一份用例均可回溯至来源文档（PRD 段落、Figma 节点），需求变更时自动识别受影响用例。
- **统一测试管理**：提供从项目、模块、用例、计划到执行的一站式管理，支持团队协作与质量度量。

### 1.3 用户价值

| 用户角色   | 核心价值                                                     |
| :--------- | :----------------------------------------------------------- |
| 测试工程师 | 告别从零编写用例，快速获得高质量草稿；一键关联需求，变更无忧。 |
| 测试组长   | 标准化用例库，评审效率提升；智能分析用例质量与需求覆盖缺口。 |
| 产品经理   | 设计即测试，Figma/PRD 完成后自动沉淀测试资产，减少沟通成本。 |
| 研发负责人 | 质量左移，需求阶段即介入测试；实时掌控项目质量风险。         |

------

## 2. 用户角色与权限

| 角色名称   | 权限范围                                                     |
| :--------- | :----------------------------------------------------------- |
| 超级管理员 | 系统全局配置、租户管理、模型提供商配置、用户管理、审计日志查看 |
| 项目管理员 | 创建/编辑/归档项目、管理项目成员、维护知识库、发起 AI 生成任务、查看项目级报告 |
| 测试工程师 | 用例库的增删改查、草稿确认/拒绝、测试计划执行、结果录入      |
| 访客       | 只读访问项目仪表盘、用例库、报告（不可编辑）                 |

> 项目内角色可进一步细分（如模块负责人），但 MVP 阶段仅做项目级权限。

## 3. 产品范围与边界

**包含功能**：

- 项目管理（项目、模块、项目配置）
- 知识库构建（PRD 解析、Figma 集成、API 规范导入）
- AI 用例生成（RAG 检索、大模型生成、草稿管理）
- 测试用例管理（增删改查、版本历史、导入导出、需求追溯）
- 测试计划与执行（计划编排、任务分配、结果录入）
- 质量报告（用例覆盖率、执行通过率、需求追溯矩阵）
- 系统管理（用户、权限、模型配置）

**不包含功能**：

- 自动化测试执行引擎（仅管理用例与结果，不执行脚本）
- 缺陷管理（可对接 Jira/TAPD 等第三方，预留 API）
- CI/CD 集成（预留 Webhook，由用户自行对接）

## 4. 功能模块详细设计

### 4.1 项目管理

#### 4.1.1 项目空间

- 每个项目拥有独立的工作区，包含仪表盘、知识库、用例库、计划、报告等导航。
- **项目仪表盘**：
  - 顶部卡片展示关键指标：用例总数、执行通过率、需求覆盖度、AI 生成用例数。
  - 趋势图：近 7 天用例执行通过率变化。
  - 最近活动：最新生成的草稿、最新执行结果。
  - 需求覆盖环形图：已覆盖需求 vs 未覆盖需求（需求来源知识库标题）。

#### 4.1.2 模块管理

- 模块以树形结构组织（如“登录/注册/订单”）。
- 每个用例必须归属于一个模块，模块名称在项目内唯一。
- 支持模块负责人设置，用于任务分配。

#### 4.1.3 项目配置管理（新增亮点）

- 项目管理员可在“项目设置”中维护键值对配置，灵活扩展项目级参数。
- **典型配置项**：
  - AI 生成参数（默认模型、温度、top_p）。
  - 环境变量（测试环境 URL、数据库连接串）。
  - 通知设置（邮件、钉钉、企业微信）。
- 配置以 JSONB 格式存储，支持复杂结构。
- 支持配置版本历史（后续迭代）。

**交互**：

- 配置列表页：展示所有配置键、值（截断显示）、最后更新时间。
- 新增/编辑：输入键（字母数字下划线）、值（可输入 JSON 或文本）、描述。
- 支持导入/导出配置（JSON 格式）。

### 4.2 知识库管理

#### 4.2.1 PRD 文档管理

**功能**：

- 上传 Word/PDF/Markdown 格式的 PRD（支持拖拽）。
- 系统自动解析文档结构（按标题分块），展示解析进度。
- 解析完成后，可在知识库列表查看文档，并展开查看分块内容。
- 支持重新解析（文档更新后）。

**交互**：

- 上传后显示“解析中”，完成后状态变为“已解析”。
- 点击文档名称进入详情页，左侧显示文档大纲，右侧显示分块列表，分块可手动合并/拆分（高级功能）。

#### 4.2.2 Figma 设计稿集成

**功能**：

1. **连接配置**：
   - 支持两种认证方式：个人访问令牌（手动输入）和 OAuth 2.0（跳转授权）。
   - 测试连接成功后，可保存配置。
2. **导入文件**：
   - 输入 Figma 文件 URL（如 `https://www.figma.com/file/xxx/项目名称`），自动提取 `file_key`。
   - 支持选择特定页面（Canvas）导入，避免全量。
3. **解析内容**：
   - 系统后台遍历节点树，提取所有文本节点及组件名称，生成节点树预览。
   - 用户可勾选哪些节点需要进入知识库（默认全选）。
   - 支持为节点添加自定义标签（如“登录页”、“表单”），便于检索过滤。
4. **同步策略**：
   - 手动触发“同步更新”：重新拉取最新文件，增量更新节点（新增插入，删除标记）。
   - 可选 Webhook 自动同步（需 Figma 企业版）。

#### 4.2.3 API 规范导入

- 支持 OpenAPI 3.0 JSON/YAML 文件上传。
- 解析出接口路径、方法、参数、响应结构，作为测试用例生成的上下文。
- 每个接口可生成对应的 API 测试用例草稿。

### 4.3 AI 用例生成

#### 4.3.1 生成任务发起

**入口**：项目空间 → AI 助手 → 新建生成任务。

**参数配置**：

- **目标模块**：必选，指定生成用例归属的模块。
- **需求描述**：自然语言输入，例如“为支付功能设计用例，包含优惠券使用场景”（可选，若为空则系统根据模块名称自动检索）。
- **文档过滤**：可选择来源文档类型（PRD / Figma / API Spec / 全部）。
- **用例数量**：5/10/20 条（默认 10）。
- **高级选项**：
  - 场景类型：正向 / 异常 / 边界（默认全选）。
  - 优先级偏好：P0~P3（默认 P2）。
  - 生成模式：常规 / 深度覆盖（深度覆盖将生成更多边界场景，耗时稍长）。

**交互**：

- 点击“立即生成”，返回任务 ID，系统在后台异步处理。
- 用户可离开页面，任务完成后通过站内信/邮件通知（需配置）。

#### 4.3.2 草稿箱与用例确认

**入口**：左侧导航 → 草稿箱（数字角标提示未处理草稿数量）。

**列表视图**：

- 按项目/模块/生成时间分组。
- 每条草稿展示：标题、生成来源（PRD 标题/Figma 页面）、置信度（高/中/低）、生成时间。
- 支持批量操作（批量确认、批量拒绝）。

**详情确认页**（核心交互）：

- **左侧**：用例字段编辑区
  - 标题（可编辑）
  - 前置条件（数组编辑器，可增删行）
  - 测试步骤（可拖拽排序，支持插入新步骤）
  - 预期结果（富文本或结构化对象）
  - 用例类型、优先级（下拉选择）
- **右侧**：引用来源展示
  - 展示该用例所引用的知识块原文（可切换多个来源）。
  - 点击“查看原文”可跳转到文档原文位置（如 Figma 节点或 PRD 段落）。
- **底部按钮**：
  - **确认并转为正式用例**：将草稿转为正式用例，自动生成用例编号（规则：`项目前缀-模块-年月日-序列`），并记录引用块。
  - **保存修改**：暂存编辑内容，不转正。
  - **拒绝并反馈原因**：弹出选项（单选：重复生成/与需求无关/质量过低/其他），填写反馈后草稿状态置为 rejected。

#### 4.3.3 生成记录与效能分析

- 提供“生成任务”历史列表，查看每次任务产出的用例数量、确认率、平均编辑耗时。
- 项目管理员可查看团队 AI 效能看板：确认率趋势、拒绝原因分布、Top 高频引用文档。

### 4.4 测试用例管理

#### 4.4.1 用例库

**功能**：

- 树状展示（项目→模块→用例），支持搜索、筛选（类型、优先级、创建人、状态）。
- 用例详情页展示完整信息，并显示“需求追溯”标签（如“来源于 PRD 2.3 节”），点击可跳转至知识库对应片段。
- 支持用例版本历史：每次更新自动生成快照，可对比差异、回滚。
- 导入导出：Excel/CSV 格式，与 Xray、TestLink 等工具对齐。

**交互**：

- 列表行操作：编辑、复制、删除、查看历史。
- 批量操作：批量修改模块/优先级/类型。

#### 4.4.2 用例评审（可选）

- 支持发起用例评审，邀请评审人，在线评论与打回。
- 评审通过后用例锁定为“已评审”状态。

### 4.5 测试计划与执行

#### 4.5.1 测试计划

**功能**：

- 创建计划：填写名称、选择关联模块、计划时间。
- **智能推荐**：根据本次迭代的需求变更（可手动标记需求 ID）自动推荐需执行的用例子集。
- 从用例库中手动添加/移除用例。
- 计划状态流转：草稿 → 进行中 → 已完成 → 已归档。

#### 4.5.2 执行跟踪

- 支持将计划中的用例分配给具体执行人。
- 执行人登录后可在“我的任务”中查看待执行用例。
- 执行结果录入：
  - 选择状态：通过/失败/阻塞/跳过。
  - 附加信息：执行日志 URL、截图、失败原因。
- 执行历史：每个用例的每次执行记录均可追溯。

### 4.6 结果分析与报告

#### 4.6.1 质量仪表盘

- **项目级**：
  - 用例通过率趋势图（按天/周）。
  - 模块缺陷分布饼图。
  - 需求覆盖度：已覆盖用例的需求数 / 总需求数。
- **团队级**：
  - 各成员执行效率（用例数/天）。
  - AI 生成用例缺陷发现率对比（对比手工用例）。

#### 4.6.2 需求追溯矩阵

- 自动生成“需求 ↔ 用例”双向追踪矩阵，支持导出 Excel。
- 需求覆盖率不足时，标记缺失用例的模块。

### 4.7 系统管理

#### 4.7.1 用户管理

- 用户列表（增删改查、重置密码）。
- 角色分配（全局角色：超级管理员/普通用户）。

#### 4.7.2 租户配置

- 多租户模式：每个租户独立项目、用户、知识库。
- 超级管理员可创建租户，分配租户管理员。

#### 4.7.3 模型提供商配置

- 支持配置多个大模型提供商（OpenAI、DeepSeek、阿里云等）。
- 设置 API Key、Endpoint、默认模型（如 `gpt-4`、`qwen-max`）。
- 支持 embedding 模型选择（维度需与 Milvus 配置一致）。

#### 4.7.4 审计日志

- 记录关键操作：文档上传、生成任务发起、草稿确认、用例删除等。
- 支持按时间、操作用户、操作类型筛选。

  - 环境变量（测试环境 URL、数据库连接串）。
  - 通知设置（邮件、钉钉、企业微信）。
- 配置以 JSONB 格式存储，支持复杂结构。
- 支持配置版本历史（后续迭代）。

**交互**：

- 配置列表页：展示所有配置键、值（截断显示）、最后更新时间。
- 新增/编辑：输入键（字母数字下划线）、值（可输入 JSON 或文本）、描述。
- 支持导入/导出配置（JSON 格式）。

### 4.2 知识库管理

#### 4.2.1 PRD 文档管理

**功能**：

- 上传 Word/PDF/Markdown 格式的 PRD（支持拖拽）。
- 系统自动解析文档结构（按标题分块），展示解析进度。
- 解析完成后，可在知识库列表查看文档，并展开查看分块内容。
- 支持重新解析（文档更新后）。

**交互**：

- 上传后显示“解析中”，完成后状态变为“已解析”。
- 点击文档名称进入详情页，左侧显示文档大纲，右侧显示分块列表，分块可手动合并/拆分（高级功能）。

#### 4.2.2 Figma 设计稿集成

**功能**：

1. **连接配置**：
   - 支持两种认证方式：个人访问令牌（手动输入）和 OAuth 2.0（跳转授权）。
   - 测试连接成功后，可保存配置。
2. **导入文件**：
   - 输入 Figma 文件 URL（如 `https://www.figma.com/file/xxx/项目名称`），自动提取 `file_key`。
   - 支持选择特定页面（Canvas）导入，避免全量。
3. **解析内容**：
   - 系统后台遍历节点树，提取所有文本节点及组件名称，生成节点树预览。
   - 用户可勾选哪些节点需要进入知识库（默认全选）。
   - 支持为节点添加自定义标签（如“登录页”、“表单”），便于检索过滤。
4. **同步策略**：
   - 手动触发“同步更新”：重新拉取最新文件，增量更新节点（新增插入，删除标记）。
   - 可选 Webhook 自动同步（需 Figma 企业版）。

#### 4.2.3 API 规范导入

- 支持 OpenAPI 3.0 JSON/YAML 文件上传。
- 解析出接口路径、方法、参数、响应结构，作为测试用例生成的上下文。
- 每个接口可生成对应的 API 测试用例草稿。

### 4.3 AI 用例生成

#### 4.3.1 生成任务发起

**入口**：项目空间 → AI 助手 → 新建生成任务。

**参数配置**：

- **目标模块**：必选，指定生成用例归属的模块。
- **需求描述**：自然语言输入，例如“为支付功能设计用例，包含优惠券使用场景”（可选，若为空则系统根据模块名称自动检索）。
- **文档过滤**：可选择来源文档类型（PRD / Figma / API Spec / 全部）。
- **用例数量**：5/10/20 条（默认 10）。
- **高级选项**：
  - 场景类型：正向 / 异常 / 边界（默认全选）。
  - 优先级偏好：P0~P3（默认 P2）。
  - 生成模式：常规 / 深度覆盖（深度覆盖将生成更多边界场景，耗时稍长）。

**交互**：

- 点击“立即生成”，返回任务 ID，系统在后台异步处理。
- 用户可离开页面，任务完成后通过站内信/邮件通知（需配置）。

#### 4.3.2 草稿箱与用例确认

**入口**：左侧导航 → 草稿箱（数字角标提示未处理草稿数量）。

**列表视图**：

- 按项目/模块/生成时间分组。
- 每条草稿展示：标题、生成来源（PRD 标题/Figma 页面）、置信度（高/中/低）、生成时间。
- 支持批量操作（批量确认、批量拒绝）。

**详情确认页**（核心交互）：

- **左侧**：用例字段编辑区
  - 标题（可编辑）
  - 前置条件（数组编辑器，可增删行）
  - 测试步骤（可拖拽排序，支持插入新步骤）
  - 预期结果（富文本或结构化对象）
  - 用例类型、优先级（下拉选择）
- **右侧**：引用来源展示
  - 展示该用例所引用的知识块原文（可切换多个来源）。
  - 点击“查看原文”可跳转到文档原文位置（如 Figma 节点或 PRD 段落）。
- **底部按钮**：
  - **确认并转为正式用例**：将草稿转为正式用例，自动生成用例编号（规则：`项目前缀-模块-年月日-序列`），并记录引用块。
  - **保存修改**：暂存编辑内容，不转正。
  - **拒绝并反馈原因**：弹出选项（单选：重复生成/与需求无关/质量过低/其他），填写反馈后草稿状态置为 rejected。

#### 4.3.3 生成记录与效能分析

- 提供“生成任务”历史列表，查看每次任务产出的用例数量、确认率、平均编辑耗时。
- 项目管理员可查看团队 AI 效能看板：确认率趋势、拒绝原因分布、Top 高频引用文档。

### 4.4 测试用例管理

#### 4.4.1 用例库

**功能**：

- 树状展示（项目→模块→用例），支持搜索、筛选（类型、优先级、创建人、状态）。
- 用例详情页展示完整信息，并显示“需求追溯”标签（如“来源于 PRD 2.3 节”），点击可跳转至知识库对应片段。
- 支持用例版本历史：每次更新自动生成快照，可对比差异、回滚。
- 导入导出：Excel/CSV 格式，与 Xray、TestLink 等工具对齐。

**交互**：

- 列表行操作：编辑、复制、删除、查看历史。
- 批量操作：批量修改模块/优先级/类型。

#### 4.4.2 用例评审（可选）

- 支持发起用例评审，邀请评审人，在线评论与打回。
- 评审通过后用例锁定为“已评审”状态。

### 4.5 测试计划与执行

#### 4.5.1 测试计划

**功能**：

- 创建计划：填写名称、选择关联模块、计划时间。
- **智能推荐**：根据本次迭代的需求变更（可手动标记需求 ID）自动推荐需执行的用例子集。
- 从用例库中手动添加/移除用例。
- 计划状态流转：草稿 → 进行中 → 已完成 → 已归档。

#### 4.5.2 执行跟踪

- 支持将计划中的用例分配给具体执行人。
- 执行人登录后可在“我的任务”中查看待执行用例。
- 执行结果录入：
  - 选择状态：通过/失败/阻塞/跳过。
  - 附加信息：执行日志 URL、截图、失败原因。
- 执行历史：每个用例的每次执行记录均可追溯。

### 4.6 结果分析与报告

#### 4.6.1 质量仪表盘

- **项目级**：
  - 用例通过率趋势图（按天/周）。
  - 模块缺陷分布饼图。
  - 需求覆盖度：已覆盖用例的需求数 / 总需求数。
- **团队级**：
  - 各成员执行效率（用例数/天）。
  - AI 生成用例缺陷发现率对比（对比手工用例）。

#### 4.6.2 需求追溯矩阵

- 自动生成“需求 ↔ 用例”双向追踪矩阵，支持导出 Excel。
- 需求覆盖率不足时，标记缺失用例的模块。

### 4.7 系统管理

#### 4.7.1 用户管理

- 用户列表（增删改查、重置密码）。
- 角色分配（全局角色：超级管理员/普通用户）。

#### 4.7.2 租户配置

- 多租户模式：每个租户独立项目、用户、知识库。
- 超级管理员可创建租户，分配租户管理员。

#### 4.7.3 模型提供商配置

- 支持配置多个大模型提供商（OpenAI、DeepSeek、阿里云等）。
- 设置 API Key、Endpoint、默认模型（如 `gpt-4`、`qwen-max`）。
- 支持 embedding 模型选择（维度需与 Milvus 配置一致）。

#### 4.7.4 审计日志

- 记录关键操作：文档上传、生成任务发起、草稿确认、用例删除等。
- 支持按时间、操作用户、操作类型筛选。

## 8. 非功能性需求

### 8.1 性能

- 页面加载首屏时间 < 2 秒。
- 常规 API 响应 < 300 毫秒（95 分位）。
- 向量检索 P99 < 500 毫秒（千万级数据）。
- 大模型调用超时设为 15 秒，超时则降级返回错误。

### 8.2 安全

- 用户密码加密存储（bcrypt）。
- JWT 密钥定期轮换，Token 有效期 2 小时。
- 敏感信息（Figma token、API Key、项目配置中的密钥）必须加密存储（AES-256）。
- 操作日志保留 ≥180 天，可审计。

### 8.3 可用性

- 系统核心功能可用性 99.9%（计划内维护除外）。
- 支持水平扩展，无单点故障。
- 关键链路（文档解析、生成任务）采用异步队列，失败自动重试。

### 8.4 扩展性

- 支持接入多家大模型厂商，通过插件化配置切换。
- 知识库存储可切换向量数据库（Milvus / pgvector / Qdrant）。
- 提供 OpenAPI 规范，便于第三方系统集成。

## 10. 风险与应对

| 风险                         | 应对策略                                                     |
| :--------------------------- | :----------------------------------------------------------- |
| 大模型生成用例质量不稳定     | ① 建立质量评估机制，用户反馈闭环；② 提供“置信度”标签，低置信度需额外审慎；③ 允许用户自定义 Prompt 模板。 |
| Figma 解析复杂度高，节点过多 | 采用异步任务、分页加载；限制单次导入节点数 ≤ 2000，超时建议分批导入。 |
| 用户拒绝率过高，导致功能废弃 | 加强用户教育，展示生成依据；快速迭代优化 Prompt，每月发布模型模板更新。 |
| 项目配置管理可能导致配置爆炸 | 限制键名长度、提供配置模板；支持配置分组和导出导入。         |
| 数据安全与隐私合规           | 私有化部署选项；与云厂商签订数据保护协议；对上传文档自动脱敏。 |
