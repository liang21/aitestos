## 1. 产品概述

### 1.1 产品愿景

**一句话定义**：AI 驱动的智能测试用例生成与管理平台。

**愿景描述**：
Aitestos 致力于解决测试团队在用例编写环节的效率瓶颈。通过 AI 技术将 PRD、Figma 设计稿、API 规范等需求文档自动转化为可执行的测试用例，实现"需求即测试"的理念，让测试工程师专注于测试策略和缺陷发现，而非重复性的用例编写工作。

### 1.2 目标用户

| 用户群体 | 痛点 | 核心诉求 |
|---|---|---|
| 测试团队负责人 | 用例编写耗时长、人力成本高 | 提升用例编写效率，缩短测试周期 |
| 测试工程师 | 重复性工作多、需求变更追溯难 | 自动生成用例草稿，快速响应需求变化 |
| 项目经理 | 测试进度不可视、覆盖率难统计 | 实时掌握测试进度和质量指标 |

### 1.3 核心价值主张

| 价值点 | 描述 | 量化目标 |
|---|---|---|
| **效率提升** | AI 自动生成测试用例草稿 | 用例编写效率提升 60%+ |
| **质量保障** | 基于需求文档生成，确保覆盖完整 | 需求覆盖率提升 40%+ |
| **追溯能力** | 用例关联需求来源，支持变更影响分析 | 需求变更响应时间缩短 50% |
| **知识沉淀** | 文档自动解析入库，形成测试知识资产 | 文档利用率 100% |

### 1.4 产品边界

**包含范围**：
- 测试用例全生命周期管理
- AI 辅助用例生成
- 测试计划编排与执行
- 需求文档知识库

**不包含范围**：
- 自动化测试执行引擎
- 缺陷管理系统（可集成）
- CI/CD 流水线
- 代码仓库管理

### 3.2 知识库模块

#### 3.2.1 PRD 文档管理

**功能**：

- 上传 Word/PDF/Markdown 格式的 PRD（支持拖拽）。
- 系统自动解析文档结构（按标题分块），展示解析进度。
- 解析完成后，可在知识库列表查看文档，并展开查看分块内容。
- 支持重新解析（文档更新后）。

**交互**：

- 上传后显示"解析中"，完成后状态变为"已解析"。
- 点击文档名称进入详情页，左侧显示文档大纲，右侧显示分块列表，分块可手动合并/拆分（高级功能）。

#### 3.2.2 Figma 设计稿集成（详细）

**功能**：

**1. 连接配置**：
- 支持两种认证方式：个人访问令牌（手动输入）和 OAuth 2.0（跳转授权）。
- 测试连接成功后，可保存配置。

**2. 导入文件**：
- 输入 Figma 文件 URL（如 `https://www.figma.com/file/xxx/项目名称`），自动提取 `file_key`。
- 支持选择特定页面（Canvas）导入，避免全量。

**3. 解析内容**：
- 系统后台遍历节点树，提取所有文本节点及组件名称，生成节点树预览。
- 用户可勾选哪些节点需要进入知识库（默认全选）。
- 支持为节点添加自定义标签（如"登录页"、"表单"），便于检索过滤。

**4. 同步策略**：
- 手动触发"同步更新"：重新拉取最新文件，增量更新节点（新增插入，删除标记）。
- 可选 Webhook 自动同步（需 Figma 企业版）。

#### 3.2.3 API 规范导入

- 支持 OpenAPI 3.0 JSON/YAML 文件上传。
- 解析出接口路径、方法、参数、响应结构，作为测试用例生成的上下文。
- 每个接口可生成对应的 API 测试用例草稿。

#### 3.2.4 文档处理流程

```
上传文件 → 存储到 OSS → 创建 Document 记录
         ↓
    推送到消息队列
         ↓
    消费者解析文档 → 生成分块（DocumentChunk）
         ↓
    调用 Embedding API → 存储向量到 Milvus
         ↓
    更新 Document 状态为 completed
```

#### 3.2.5 文档状态

| 状态 | 说明 |
|---|---|
| `pending` | 等待处理 |
| `processing` | 正在解析 |
| `completed` | 处理完成 |
| `failed` | 处理失败 |

#### 3.2.6 文档更新与删除策略

**文档更新（重新解析）**：

- 重新解析后，已关联用例的文档块 ID 若发生变化：
  - 保留用例引用记录，标记为"源文档已变更"
  - 在用例详情页显示 ⚠️ 警告提示
  - 不自动删除或修改已确认的用例

**文档删除**：

- 采用软删除，保留引用关系
- 用例详情页显示"源文档已删除"状态
- 提供"移除引用"操作供用户手动清理

### 3.3 AI 用例生成模块（核心）

#### 3.3.1 生成任务发起

**入口**：项目空间 → AI 助手 → 新建生成任务。

**参数配置**：

| 参数 | 类型 | 必填 | 说明 |
|---|---|:---:|---|
| 目标模块 | Module ID | 是 | 用例归属模块 |
| 需求描述 | Text | 是 | 自然语言描述测试需求 |
| 用例数量 | Integer | 否 | 期望生成的用例数量，默认 5 |
| 用例类型 | Enum | 否 | 功能/性能/API/UI/安全 |

**高级选项**：
- 场景类型：正向 / 异常 / 边界（默认全选）
- 优先级偏好：P0~P3（默认 P2）
- 生成模式：常规 / 深度覆盖（深度覆盖将生成更多边界场景，耗时稍长）

**交互**：

- 点击"立即生成"，返回任务 ID，系统在后台异步处理。
- 用户可离开页面，任务完成后通过站内信/邮件通知（需配置）。

#### 3.3.2 RAG 检索流程

```
用户需求描述
     ↓
调用 Embedding API 转为向量
     ↓
在 Milvus 中检索 Top-K 相似文档块
     ↓
组装 Prompt（需求 + 检索上下文）
     ↓
调用 LLM 生成用例 JSON
     ↓
解析并存入 generated_case_draft 表
```

**检索为空的降级策略**：

| 场景 | 处理方式 |
|---|---|
| 检索结果 < 1 个文档块 | 弹窗提示"知识库内容不足，生成质量可能较低"，用户确认后继续，置信度强制标记为"低" |
| 完全无知识库 | 禁止生成任务发起，提示"请先上传需求文档" |

#### 3.3.3 AI 置信度定义

置信度用于标识 AI 生成结果的可信程度，帮助用户判断审核优先级。

| 级别 | 判定条件 | UI 展示 |
|---|---|---|
| **高** | 检索到 ≥2 个相关文档块，且语义相似度 > 0.8 | 🟢 绿色标签 |
| **中** | 检索到 1 个相关文档块，且语义相似度 0.5-0.8 | 🟡 黄色标签 |
| **低** | 无匹配文档块，纯 LLM 通用知识生成 | 🔴 红色标签 |

**语义相似度**：通过向量余弦相似度计算，存储于 `DocumentChunk` 的 embedding 字段。

#### 3.3.3 草稿箱与用例确认

**入口**：左侧导航 → 草稿箱（数字角标提示未处理草稿数量）。

**列表视图**：

- 按项目/模块/生成时间分组。
- 每条草稿展示：标题、生成来源（PRD 标题/Figma 页面）、置信度（高/中/低）、生成时间。
- 支持批量操作（批量确认、批量拒绝）。

**详情确认页**（核心交互）：

- **左侧**：用例字段编辑区
  - 标题（可编辑）
  - 前置条件（数组编辑器，可增删行）
  - 测试步骤（可拖拽排序，支持插入新步骤）
  - 预期结果（富文本或结构化对象）
  - 用例类型、优先级（下拉选择）
- **右侧**：引用来源展示
  - 展示该用例所引用的知识块原文（可切换多个来源）。
  - 点击"查看原文"可跳转到文档原文位置（如 Figma 节点或 PRD 段落）。
- **底部按钮**：
  - **确认并转为正式用例**：将草稿转为正式用例，自动生成用例编号（规则：`项目前缀-模块-年月日-序列`），并记录引用块。
  - **保存修改**：暂存编辑内容，不转正。
  - **拒绝并反馈原因**：弹出选项（单选：重复生成/与需求无关/质量过低/其他），填写反馈后草稿状态置为 rejected。

#### 3.3.4 生成记录与效能分析

- 提供"生成任务"历史列表，查看每次任务产出的用例数量、确认率、平均编辑耗时。
- 项目管理员可查看团队 AI 效能看板：确认率趋势、拒绝原因分布、Top 高频引用文档。

#### 3.3.6 用例编号规则

**格式**：`{项目前缀}-{模块缩写}-{日期}-{序号}`

**示例**：`ECO-USR-20260402-001`

**编号组成说明**：

| 组成部分 | 来源 | 说明 |
|---|---|---|
| 项目前缀 | 创建项目时由用户输入 | 必填，2-4 位大写字母，项目内唯一 |
| 模块缩写 | 创建模块时由用户输入 | 必填，2-4 位大写字母，项目内唯一 |
| 日期 | 系统自动生成 | 格式 YYYYMMDD |
| 序号 | 系统自动递增 | 当日该模块下 001-999 递增 |

**存储要求**：

- `Project.prefix`: String，唯一索引
- `Module.abbreviation`: String，项目内唯一

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 说明 |
|---|---|---|
| API 响应时间 | P99 < 500ms | 标准查询接口 |
| 用例列表加载 | < 2s | 1000 条用例 |
| 文档解析 | < 30s | 单个 50 页 PDF |
| AI 生成任务 | < 60s | 单次生成 5 个用例 |
| 向量检索 | P99 < 500ms | 千万级数据 |

#### 4.1.1 LLM 调用超时与失败处理

**超时配置**：

| 配置项 | 超时时间 | 说明 |
|---|---|---|
| LLM 单次调用 | 30s | 大模型推理超时 |
| 总任务超时 | 120s | 含检索 + 生成全流程 |
| Embedding 调用 | 10s | 向量化超时 |

**失败处理策略**：

| 失败类型 | 处理方式 |
|---|---|
| LLM 超时 | 任务状态置为 `failed`，提示"生成超时，请重试" |
| 模型服务异常 | 记录错误日志，提示"模型服务暂时不可用，请稍后重试" |
| Embedding 失败 | 降级为不使用 RAG 增强，置信度标记为"低"并告知用户 |
| 队列积压 | 显示预计等待时间，支持任务取消 |

**重试机制**：

- 用户可对失败任务点击"重试"
- 系统不自动重试，避免资源浪费

### 4.2 安全要求

| 要求 | 描述 |
|---|---|
| 认证方式 | JWT Token，有效期 2 小时 |
| 密码存储 | bcrypt 加密 |
| 敏感配置 | 应用层加密存储（AES-256） |
| 访问控制 | 基于角色的权限控制（RBAC） |
| 审计日志 | 敏感操作记录审计日志，保留 ≥180 天 |
| Figma Token | 必须加密存储 |

#### 4.2.1 审计日志记录的操作

| 操作类别 | 具体操作 | 审计内容 |
|----------|----------|----------|
| **认证** | 登录、登出、密码修改 | 用户ID、IP、时间、结果 |
| **项目管理** | 创建、删除项目 | 操作人、项目ID、时间 |
| **配置管理** | 新增、修改、删除配置 | 操作人、配置键、旧值、新值 |

## 6. 数据指标

### 6.1 北极星指标

| 指标 | 定义 | 目标 |
|---|---|---|
| **AI 用例采用率** | 确认的草稿数 / 生成的草稿总数 | > 70% |

### 6.2 过程指标

| 指标 | 定义 | 说明 |
|---|---|---|
| 用例生成数量 | AI 生成的用例总数 | 衡量 AI 使用量 |
| 文档上传数量 | 知识库文档总数 | 衡量知识库规模 |
| 计划执行率 | 已完成计划 / 创建计划总数 | 衡量计划完成度 |
| 用例通过率 | 执行结果为通过的次数 / 总执行次数 | 衡量产品质量 |

### 6.3 效能指标

| 指标 | 定义 | 目标 |
|---|---|---|
| 平均生成时间 | 从任务提交到草稿生成的耗时 | < 60s |
| 用例覆盖率 | 已覆盖需求数 / 需求总数 | > 80% |

#### 6.3.1 需求覆盖率计算口径

**计算公式**：

```
需求覆盖率 = 已关联用例的文档块数 / 知识库文档块总数 × 100%
```

**计算规则**：

| 规则 | 说明 |
|---|---|
| 粒度 | 按 `DocumentChunk` 粒度计算 |
| 已覆盖判定 | 一个文档块被任意用例引用即视为"已覆盖" |
| 展示位置 | 项目仪表盘环形图 |
| 更新频率 | 实时计算，或每 5 分钟刷新缓存 |

**注意**：手动创建的用例也可关联文档块，计入覆盖率统计。

## 10. 风险与应对

| 风险 | 应对策略 |
|---|---|
| **大模型生成用例质量不稳定** | ① 建立质量评估机制，用户反馈闭环；② 提供"置信度"标签，低置信度需额外审慎；③ 允许用户自定义 Prompt 模板。 |
| **Figma 解析复杂度高，节点过多** | 采用异步任务、分页加载；限制单次导入节点数 ≤ 2000，超时建议分批导入。 |
| **用户拒绝率过高，导致功能废弃** | 加强用户教育，展示生成依据；快速迭代优化 Prompt，每月发布模型模板更新。 |
| **项目配置管理可能导致配置爆炸** | 限制键名长度、提供配置模板；支持配置分组和导出导入。 |
| **数据安全与隐私合规** | 私有化部署选项；与云厂商签订数据保护协议；对上传文档自动脱敏。 |
