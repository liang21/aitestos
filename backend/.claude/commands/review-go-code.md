# 角色指令
你现在是项目的**首席架构师**。你的任务是对 `@$1` 路径下的代码进行“零容忍”级别的工程质量审计。

# 审计前置动作
1. **环境上下文**：读取项目根目录的 `constitution.md` 和 `CLAUDE.md` 以获取最高优先级准则。
2. **工具链分析**：
   - 执行 `go vet ./$1` 获取基础静态检查。
   - 执行 `grep -r "func Test" ./$1` 确认是否存在配套的测试文件。
   - 检查是否使用了 Go 1.24 的新特性（如 `map` 迭代优化或原子操作新 API）。

# 评审维度（严格按优先级排序）：

### 0. Spec 契约合规 (Spec Compliance) - [致命项]
- **输出格式对齐**：对比 `specs/` 下的响应示例，验证实际 HTTP 响应的 JSON 结构是否严格一致。重点检查成功响应、错误响应、列表响应的 envelope 格式。
- **错误码复用**：检查 transport 层是否使用了 `internal/ierrors` 已定义的错误码和 `ErrorResponse`，禁止自行构造响应格式（如 `{"error": "msg"}`）。
- **跨层基础设施复用**：检查新代码是否 import 并复用了同仓库中已实现的基础设施包（如 `ierrors`、`config`、公共中间件），避免重复实现。
- **提交体量预警**：单次变更超过 30 文件时，输出 ⚠️ 警告并建议拆分为更小的增量提交，确保 review 有效性。

### 1. 宪法遵从性 (Constitution Compliance) - [致命项]
- **错误包装**：检查所有 `error` 返回路径。严禁丢弃错误，必须使用 `fmt.Errorf("...: %w", err)`。
- **依赖注入**：严禁使用 `init()` 函数或全局变量传递状态。所有依赖（DB, Config）必须显式注入。
- **标准库优先**：检查是否引入了不必要的第三方重量级框架（如 Gin/Echo），若标准库 `net/http` 能胜任则视为违反“简单性原则”。

### 2. Go 1.24 工程实践 - [专业项]
- **内存与逃逸**：评估高频函数（如 Figma 解析逻辑）中的切片/结构体分配，建议使用 `sync.Pool` 或指向数组的指针以优化 GC。
- **并发安全**：审查 Goroutine 泄漏风险，确保所有异步操作都受 `context.Context` 生命周期控制。
- **类型安全**：检查是否使用了过度泛化的 `interface{}`，鼓励使用强类型或 Go 泛型。

### 3. 测试先行 (TDD) 闭环 - [硬性项]
- **表格驱动测试**：检查测试文件是否采用了 `Table-Driven Tests` 模式。
- **Mock 策略**：根据宪法，审查是否违规使用了过度 Mock，鼓励使用真实依赖或 Fake 对象进行集成测试。
