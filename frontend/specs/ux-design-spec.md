# 第一部分：设计总纲

## 1.1 设计哲学：Precision Intelligence

Aitestos 的设计围绕一个核心理念——**Precision Intelligence**（精准智能）。

这不是一个通用的企业管理后台。它是一个以 AI 为核心驱动力的测试平台，用户在此与 AI 协同完成从需求到用例的转化。设计需要在两个维度之间找到精确的平衡：

| 维度     | 特征                                       | 体验隐喻             |
| -------- | ------------------------------------------ | -------------------- |
| 精准工作区 | 高密度信息、紧凑布局、中性色调、克制动效   | 精密仪器面板         |
| AI 智能时刻 | 品牌紫色辉光、渐变表面、有目的的动效       | AI 正在"思考"的视觉化 |

**设计不是装饰，而是功能的一部分。** 每一个视觉选择都服务于认知效率或情感引导。当用户在密集的用例列表中工作时，界面退让为中性背景，让数据成为主角；当用户与 AI 交互时，界面通过光效和动效传达"系统正在为你工作"的信号。

### 三个核心设计目标

1. **企业级精密感**：信息层级清晰，操作路径最短，数据展示紧凑有序——如同精密仪器的控制面板
2. **AI 智能感知**：AI 相关交互通过品牌色辉光、渐变表面和有目的的动效传达智能感，让用户清晰感知"AI 正在工作"
3. **效率至上**：高频操作一步到位，批量操作流畅，键盘友好，减少一切不必要的页面跳转和等待

## 1.2 设计原则

| 原则             | 说明                                                                                           |
| ---------------- | ---------------------------------------------------------------------------------------------- |
| **一致性**       | 相同功能使用相同的交互模式、视觉样式和文案。所有列表页共享统一的表格规范，所有表单共享验证规范 |
| **可预测性**     | 操作结果可预期。按钮文案清晰表达行为，危险操作需二次确认，不可逆操作标注后果                   |
| **效率优先**     | 高频路径最短。支持批量操作、快捷键、双击进入详情、内联编辑等提效手段                           |
| **渐进式复杂度** | 基础功能开箱即用，高级功能折叠隐藏（如 AI 高级选项），避免界面复杂度过高                       |
| **数据驱动**     | 关键指标前置展示（仪表盘），状态色彩编码直观，趋势图表辅助决策                                 |
| **可访问性**     | 满足 WCAG 2.1 AA 标准，色彩对比度达标，支持键盘导航，语义化 HTML，尊重 `prefers-reduced-motion` |

## 2.2 字体排版

### 字体族

| 用途             | 字体栈                                                                   | 说明                                   |
| ---------------- | ------------------------------------------------------------------------ | -------------------------------------- |
| 标题 / 英文正文  | `'DM Sans', system-ui, -apple-system, sans-serif`                        | 几何无衬线体，辨识度高，字重变化丰富   |
| 中文正文         | `system-ui, -apple-system, 'PingFang SC', 'Microsoft YaHei', sans-serif` | 系统中文字体，保证渲染一致性           |
| 等宽（用例编号） | `'JetBrains Mono', 'Fira Code', Consolas, ui-monospace, monospace`       | 编程等宽字体，连字支持，数据表格首选   |

> **DM Sans** 选择理由：几何造型干净利落，x-height 适中保证小字号可读性，OpenType 特性丰富。与中文系统字体搭配自然，不会产生视觉割裂。通过 Google Fonts 引入，首屏预加载。

### 字体加载策略

为防止 FOIT（Flash of Invisible Text）和 CLS（Cumulative Layout Shift），字体加载遵循以下策略：

```html
<!-- HTML <head> 中 -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link rel="preload" as="style" href="https://fonts.googleapis.com/css2?family=DM+Sans:wght@400;500;600&family=JetBrains+Mono:wght@400&display=swap">
```

| 策略              | 值                          | 说明                                     |
| ----------------- | --------------------------- | ---------------------------------------- |
| `font-display`    | `swap`                      | 优先使用系统字体渲染，字体加载完成后替换 |
| `preconnect`      | Google Fonts 域名           | 提前建立 DNS + TLS 连接                  |
| `preload`         | 仅加载 wght 400/500/600     | 预加载关键字重，不加载全部变体           |
| 字体子集          | `latin` + `latin-ext`       | 仅加载拉丁字符子集，减小文件体积         |
| CLS 防护          | `font-size-adjust: 0.5`     | DM Sans 与 system-ui 的 x-height 对齐    |

| 字重    | 值  | 使用场景                       |
| ------- | --- | ------------------------------ |
| Regular | 400 | 正文、描述、辅助文字           |
| Medium  | 500 | 小标题、按钮文字、表头、标签   |
| Bold    | 600 | 页面大标题、统计数值、强调文字 |

### 数字排版规则

- **数据表格**：`font-variant-numeric: tabular-nums`，确保数字等宽对齐
- **统计数值**：`font-feature-settings: 'tnum'`，大字号（28-36px）时字重 600
- **编号列**：等宽字体 + `letter-spacing: -0.02em`，保持紧凑但可读

### 节奏间距（Rhythmic Spacing）

> 通过间距层级的戏剧性对比，建立清晰的视觉节奏。页面级区块使用大间距（3xl-4xl），卡片内部保持紧凑（base-lg），让用户直觉感知信息层级。

| 层级         | 间距 | 示例                               |
| ------------ | ---- | ---------------------------------- |
| 页面区块之间 | 3xl+ | 统计卡片区与趋势图区之间           |
| 卡片之间     | lg   | 统计卡片行内间距                   |
| 卡片内部     | base | 卡片标题与内容之间                 |

### AI 专属动效详解

#### 内容揭示动画（Content Reveal）

AI 生成的内容（草稿列表、建议文本）出现时使用统一的揭示动画：

```css
@keyframes ai-reveal {
  from {
    opacity: 0;
    transform: translateY(8px) scale(0.97);
    filter: blur(2px);
  }
  to {
    opacity: 1;
    transform: translateY(0) scale(1);
    filter: blur(0);
  }
}
```

- 时长：400ms
- 缓动：ease-out
- 交错延迟：列表项之间 80ms 递增

#### 辉光脉冲（Glow Pulse）

processing 状态的元素使用辉光脉冲动画：

```css
@keyframes glow-pulse {
  0%, 100% { box-shadow: 0 0 15px rgba(123,97,255,0.15); }
  50% { box-shadow: 0 0 25px rgba(123,97,255,0.30); }
}
```

- 时长：2s
- 缓动：ease-in-out
- 无限循环

#### 进度闪烁条（Progress Shimmer）

加载中的骨架屏和进度条使用渐变扫描效果：

```css
@keyframes shimmer {
  0% { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}
/* 背景：linear-gradient(90deg, transparent 0%, rgba(123,97,255,0.08) 50%, transparent 100%) */
```

- 时长：1.5s
- 缓动：linear
- 无限循环

## 4.3 核心用户任务流程（User Task Flow）

> 以下流程描述用户从起点到终点的完整操作路径，标注涉及的页面、API 和状态流转。

### 流程 1：AI 生成用例全流程

```
知识库为空？
  ├── 是 → [知识库页] 上传 PRD 文档 → 等待解析(completed) ──┐
  └── 否 ─────────────────────────────────────────────────────┘
                                                              ↓
[AI 生成页] 新建生成任务 → 填写需求描述 + 选模块 → 提交
        ↓ POST /generation/tasks
[任务详情页] 等待处理(processing) → 轮询每 5s
        ↓ GET /generation/tasks/{id}
    ┌─── completed ───┐
    │                  │
    ▼                  ▼
[草稿列表]        [任务失败] → 重试 → 回到等待
  │ 查看每条草稿
  ▼
[草稿确认页] 编辑内容 → 查看引用来源
  ├── 确认 → POST /generation/drafts/{id}/confirm → [用例详情页]
  ├── 拒绝 → POST /generation/drafts/{id}/reject → 回到草稿列表
  └── 保存 → 暂存修改
```

**涉及页面**：知识库文档列表、文档上传 Modal、AI 生成任务列表、新建生成任务、任务详情、草稿确认
**涉及 API**：POST /knowledge/documents → GET /knowledge/documents/{id} → POST /generation/tasks → GET /generation/tasks/{id}（轮询）→ GET /generation/tasks/{id}/drafts → POST /generation/drafts/{id}/confirm

### 流程 2：测试执行全流程

```
[计划列表页] 新建计划
        ↓
[新建计划页] 填写名称 + 选择用例 → 提交
        ↓ POST /plans
[计划详情页] 状态: draft
  │ 点击"开始执行"
  ▼ POST /plans/{id}/status { status: active }
[计划详情页] 状态: active
  │ 逐条录入执行结果
  │   ├── 快捷录入: 内联 Select → Toast 确认
  │   └── 详细录入: 点击"录入" → Modal → 提交
  │ POST /plans/{id}/results
  │
  │ 全部执行完毕 → 点击"标记完成"
  ▼ POST /plans/{id}/status { status: completed }
[计划详情页] 状态: completed → 查看统计（通过率、失败数）
  │ 可选: 点击"归档"
  ▼ POST /plans/{id}/status { status: archived }
```

**涉及页面**：计划列表、新建计划、计划详情
**涉及 API**：POST /plans → GET /plans/{id} → PATCH /plans/{id}/status（3 次）→ POST /plans/{id}/results（多次）

### 流程 3：需求变更响应流程

```
[知识库页] 发现文档需更新
  │ 点击"重新解析"（或重新上传）
  ▼ POST /knowledge/documents（覆盖）
[文档详情页] 等待重新解析 → completed
  │ 文档块 ID 可能变化
  ▼
[用例详情页] 查看已关联用例
  │ AI 元数据区显示 ⚠️ "源文档已变更"
  │ 用户手动核验用例内容
  │   ├── 内容仍有效 → 忽略警告
  │   └── 内容需更新 → 编辑用例
  ▼ PUT /testcases/{id}
```

**涉及页面**：知识库列表、文档详情、用例详情
**涉及 API**：POST /knowledge/documents → GET /knowledge/documents/{id}/chunks → GET /testcases/{id} → PUT /testcases/{id}

## 6.3 全局状态与路由守卫

### Zustand Store 结构

```typescript
interface AppStore {
  // 用户
  user: { id: string; username: string; email: string; role: string } | null
  // 当前项目
  currentProject: { id: string; name: string; prefix: string } | null
  // UI 状态
  sidebarCollapsed: boolean
  // 通知
  pendingDraftCount: number
  notificationCount: number
}
```

### Token 管理

- 存储：localStorage `token` + `refresh_token`
- 拦截器：Axios request interceptor 自动附加 `Authorization: Bearer {token}`
- 刷新：401 响应时调用 `POST /auth/refresh`，成功则重试原请求，失败则清除 token 并跳转 `/login`

### 路由守卫

| 条件                    | 行为                          |
| ----------------------- | ----------------------------- |
| 未登录 + 访问需认证页面 | 重定向到 `/login`             |
| 已登录 + 访问 `/login`  | 重定向到 `/projects`          |
| 401 API 响应            | 清除 token，重定向到 `/login` |

### API 错误处理映射

| HTTP 状态码 | 处理方式     | UI 反馈                                     | 恢复指引（Recovery）                              |
| ----------- | ------------ | ------------------------------------------- | ------------------------------------------------- |
| 400         | 表单验证提示 | 对应字段下方红色错误提示                    | 指出具体哪个字段不合法 + 正确格式说明             |
| 401         | 跳转登录页   | 清除 token + redirect `/login`              | 登录页 Alert："会话已过期，请重新登录"            |
| 403         | 无权限提示   | Notification.warning "无操作权限"           | "请联系管理员 [项目设置]" 附带跳转链接            |

## 13.3 动效降级

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

- AI 揭示动画降级为瞬间出现
- 辉光脉冲动画完全停止
- Shimmer 效果降级为静态灰色
- 页面过渡降级为瞬间切换

- 核心操作流程（用例管理、AI 生成、测试执行）在 Desktop 端完整可用
- 响应式适配优先级：Desktop > Laptop > Tablet > Mobile
- **尊重 `prefers-reduced-motion`**：所有动画必须有静态降级方案
